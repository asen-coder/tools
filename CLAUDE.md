# tools — 个人工具集合

每个一级子目录是一个完全独立的工具，技术栈各异，互不依赖。本仓库只是收纳盒，
不做统一构建、不做跨工具的依赖管理。

## 铁律（防止 AI 串味）

1. 工具之间禁止互相 import / 引用代码。需要相同逻辑就各自拷贝一份，不抽公共库。
2. 处理某个工具时，只读写该工具自己的目录，不要参考、套用其他工具的实现。
3. 每个工具有自己的 CLAUDE.md，那才是该工具的权威上下文；根目录这份只管全局约定。

## 目录约定

- 一级子目录 = 一个工具，目录名即工具名。
- 下划线开头的目录（如 `_template`）不是工具，是仓库级辅助资源。
- 工具内部结构由各自的 CLAUDE.md 说明。
- 新增工具：复制 `_template/` 内容到新目录，按实际填写其 CLAUDE.md，并在根 README 索引表加一行。

## 协作习惯

改某个工具时，在对话里明确说"只动 `<工具名>`"，或直接在该工具目录下启动，
以收窄上下文检索范围。
